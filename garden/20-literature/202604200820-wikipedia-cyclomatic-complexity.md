# Цикломатическая сложность

Источник: https://ru.wikipedia.org/wiki/Цикломатическая_сложность

Постоянная версия: https://ru.wikipedia.org/w/index.php?title=Цикломатическая_сложность&oldid=146366923

Дата обращения: 2026-04-20

## Ключевые идеи

- Цикломатическая сложность - структурная метрика кода, построенная по графу потока управления, а не оценка времени выполнения или алгоритмической сложности.
- Узлы графа соответствуют неделимым блокам команд, рёбра - возможным переходам управления между блоками.
- Базовая формула для графа потока управления: `M = E - N + 2P`, где `E` - число рёбер, `N` - число узлов, `P` - число компонент связности. Для одной функции или метода обычно `P = 1`.
- В альтернативной формулировке, где точки выхода соединяют с точкой входа, используется `M = E - N + P`; это число независимых циклов в расширенном графе.
- Для структурированной программы с одной точкой входа и одной точкой выхода значение можно понимать проще: число точек ветвления плюс один.
- Практический смысл метрики - оценить, сколько независимых путей нужно удерживать в голове, тестировать и сопровождать.
- Маккейб предлагал ограничивать сложность модулей и дробить модуль, если сложность превышает 10; NIST включал этот подход в методику структурного тестирования с оговоркой, что иногда лимит можно ослаблять до 15 или превышать при письменном обосновании.
- Для тестирования `M` полезна как оценка усилия: верхняя оценка числа тестов для покрытия условий и нижняя оценка числа тестов для полного покрытия путей.
- Цикломатическая сложность также входит в состав более широких метрик сопровождаемости, например maintainability index.

## Что забрать в permanent notes

- Цикломатическая сложность считает число линейно независимых путей через граф управления, а не "сложность алгоритма".
- Значение `M` помогает оценить минимальный масштаб тестирования, но не заменяет анализ данных, рисков и бизнес-правил.
- Порог сложности полезен как инженерный guardrail: при превышении стоит разделить код или явно объяснить, почему сложность оправдана.

## Цитаты

- `M = E - N + 2P`
- "число линейно независимых маршрутов"
