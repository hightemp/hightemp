# Garden

Структура Zettelkasten для атомарных заметок и digital garden.

## Папки

- `00-inbox/` - быстрый сбор идей до разбора.
- `10-fleeting/` - сырые мысли, временные наблюдения, черновые вопросы.
- `20-literature/` - заметки по книгам, статьям, докладам, подкастам и другим материалам.
- `30-permanent/` - атомарные постоянные заметки, написанные своими словами.
- `40-moc/` - карты содержания, индексы, тематические хабы и навигационные заметки.
- `50-sources/` - карточки источников и reference packs: библиография, ссылки, короткие выдержки, таблицы, сравнения, сниппеты, команды и другая source-specific опора.
- `60-projects/` - проектные заметки, которые используют постоянные заметки.
- `90-templates/` - шаблоны заметок.
- `assets/` - изображения, PDF, схемы и вложения.

## Имена файлов

Для атомарных заметок используйте стабильные ID:

```text
YYYYMMDDHHMM-short-title.md
```

Пример:

```text
202604191530-feedback-loops-in-learning.md
```

## Процесс

1. Быстро фиксируйте входящие идеи в `00-inbox/`.
2. Переносите сырые идеи в `10-fleeting/`, если им нужно время на обдумывание.
3. Заметки по источникам храните в `20-literature/`, а source-specific материалы, reference packs и опорные артефакты - в `50-sources/`.
4. Выжимайте одну самостоятельную идею в одну заметку в `30-permanent/`.
5. Связывайте постоянные заметки через тематические карты в `40-moc/`.

## Source Packs

Если у источника есть полезные таблицы, сравнения, команды, кодовые сниппеты или куски текста, которые хочется сохранить ближе к оригиналу, не создавайте отдельные top-level папки вроде `tables/` или `snippets/`.

Используйте такой принцип:

- `20-literature/` - ваша переработанная заметка по источнику
- `50-sources/` - source pack с сырыми или полуобработанными артефактами из этого источника
- `assets/` - скриншоты, PDF, CSV, изображения и другие вложения

Удобный паттерн:

```text
garden/20-literature/202604291200-some-source.md
garden/50-sources/202604291200-some-source-reference-pack.md
garden/assets/202604291200-some-source/
```

## Ссылки

Используйте обычные относительные Markdown-ссылки, чтобы заметки открывались в VS Code и GitHub:

```markdown
[Название заметки](../30-permanent/202604191530-note-title.md)
```
