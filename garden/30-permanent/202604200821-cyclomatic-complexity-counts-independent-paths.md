# Цикломатическая сложность считает независимые пути управления

Коротко: цикломатическая сложность показывает, сколько линейно независимых маршрутов существует через код по его графу управления.

## Смысл

Цикломатическая сложность не про скорость алгоритма и не про размер данных. Это топологическая метрика: она смотрит на форму управляющего потока. Если в коде нет ветвлений и циклов, через него есть один путь. Каждый `if`, условный цикл, `case` или другой разветвитель добавляет альтернативы, которые нужно понимать отдельно.

Практический смысл метрики в том, что она делает ветвистость кода видимой числом. Для графа потока управления используется формула `M = E - N + 2P`: рёбра минус узлы плюс поправка на число независимых компонент. Для обычной функции или метода `P` обычно равно `1`, поэтому рост `M` чаще всего соответствует росту числа решений внутри кода.

Это число не объясняет всю сложность программы. Оно не видит сложность предметной области, имена, зависимости, изменяемое состояние или неудачные абстракции. Зато оно хорошо подсвечивает место, где в одном фрагменте кода накопилось много независимых маршрутов.
