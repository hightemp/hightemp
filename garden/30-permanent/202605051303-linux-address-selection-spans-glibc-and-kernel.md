# Выбор адреса в Linux распределен между glibc и ядром

Коротко: в Linux порядок адресов назначения формирует glibc, а выбор адреса источника зависит от ядра и текущего routing state.

## Смысл

Выбор адреса при подключении нельзя полностью объяснить ни DNS, ни одним конфигом. В Linux эта работа разделена: `getaddrinfo` в glibc сортирует адреса назначения, но для оценки кандидатов ему важно понять, какой локальный адрес источника подойдет для каждого адреса назначения. Это уже связано с ядром, маршрутами, интерфейсами и состоянием машины в момент запроса.

Поэтому совместимость с "системным резолвером" означает больше, чем повторить RFC в коде. Нужно учитывать фактическое поведение glibc и ядра вместе. Если runtime или приложение реализует только часть этого поведения, оно может получить другой порядок адресов и другие сетевые последствия.
