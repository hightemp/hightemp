# `/etc/gai.conf` настраивает адресную политику, а не все IPv4/IPv6 поведение

Коротко: `/etc/gai.conf` меняет таблицы `label` и `precedence` для `getaddrinfo`, но не является универсальным переключателем выбора адресов.

## Смысл

`/etc/gai.conf` полезен, когда нужно изменить политику сортировки адресов назначения: например, поменять приоритет IPv4/IPv6 или задать другие `label` для локальных IPv6-префиксов. Но этот файл не управляет всей логикой соединения. Он не заменяет маршрутизацию, не исправляет недоступный IPv6 и не выбирает адрес источника сам по себе.

Правильная ментальная модель: `/etc/gai.conf` - это вход в одну часть системной политики, а не главный рубильник сети. Если проблема зависит от source address selection, routing state или поведения runtime-резолвера, менять только `gai.conf` может быть недостаточно.
