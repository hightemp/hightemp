**1. Диагностика сети:**

*   **Проверка доступности и соединения:**
    *   `ping` - Отправляет ICMP Echo Request пакеты на указанный хост для проверки доступности и измерения времени отклика.
    *   `traceroute` / `tracepath` - Отображает маршрут следования пакетов до указанного хоста, показывая промежуточные узлы и время отклика на каждом.
    *   `mtr` - Комбинирует функциональность `ping` и `traceroute`, отображая статистику потерь и задержек на каждом узле маршрута в реальном времени.
    *   `arping` - Отправляет ARP-запросы для определения доступности хоста в локальной сети по MAC-адресу.

*   **Работа с DNS:**
    *   `nslookup` - Интерактивный инструмент для запросов к DNS-серверам. Позволяет получать различные типы записей (A, MX, CNAME и т.д.).
    *   `dig` (Domain Information Groper) - Более продвинутый инструмент для запросов к DNS-серверам, предоставляющий подробную информацию о DNS-ответах.
    *   `host` - Простая утилита для преобразования доменных имен в IP-адреса и наоборот.

*   **Анализ сетевого трафика и соединений:**
    *   `tcpdump` - Мощный инструмент для перехвата и анализа сетевого трафика. Позволяет фильтровать трафик по различным критериям (протокол, порт, IP-адрес и т.д.) и сохранять его в файл для последующего анализа.
    *   `wireshark` - Графический анализатор сетевого трафика, предоставляющий удобный интерфейс для просмотра и анализа перехваченных пакетов.
    *   `tshark` - Консольная версия Wireshark.
    *   `netstat` / `ss` - Отображают информацию о сетевых соединениях, таблицах маршрутизации, статистике интерфейсов и открытых портах. `ss` является более современной и рекомендуемой альтернативой `netstat`.
    *   `lsof` - Показывает список открытых файлов и связанных с ними процессов, включая сетевые соединения.

*   **Сканирование портов и служб:**
    *   `nmap` (Network Mapper) - Мощный инструмент для сканирования портов, обнаружения сетевых служб, определения операционных систем и многого другого.
    *   `nc` / `netcat` - "Швейцарский нож" для работы с сетевыми соединениями. Позволяет устанавливать соединения, передавать данные, прослушивать порты и многое другое.

*   **Измерение производительности сети:**
    *   `iperf` / `iperf3` - Инструмент для измерения пропускной способности сети между двумя узлами.
    *   `speedtest-cli` - Консольный клиент для популярного сервиса Speedtest.net, позволяющий измерить скорость интернет-соединения.

**2. Настройка сети:**

*   **Управление сетевыми интерфейсами:**
    *   `ifconfig` (устаревает) - Позволяет просматривать и настраивать параметры сетевых интерфейсов (IP-адрес, маска сети, шлюз и т.д.).
    *   `ip` - Современная утилита для управления сетевыми интерфейсами, маршрутизацией и другими сетевыми настройками. Заменяет `ifconfig`, `route` и другие устаревшие утилиты.
        *   `ip addr` - Управление адресами интерфейсов.
        *   `ip link` - Управление состоянием интерфейсов.
        *   `ip route` - Управление таблицей маршрутизации.
    *   `ifup` / `ifdown` - Скрипты для активации и деактивации сетевых интерфейсов, обычно используемые системой инициализации.

*   **Беспроводные сети:**
    *   `iwconfig` (устаревает) - Утилита для настройки беспроводных сетевых интерфейсов (SSID, режим работы, ключ шифрования и т.д.).
    *   `iw` - Современная утилита для управления беспроводными интерфейсами, заменяющая `iwconfig`.
    *   `wpa_supplicant` - Утилита для подключения к беспроводным сетям, защищенным с помощью WPA/WPA2/WPA3.

*   **DHCP:**
    *   `dhclient` - Клиент DHCP, используемый для автоматического получения IP-адреса и других сетевых настроек от DHCP-сервера.

*   **Межсетевые экраны:**
    *   `iptables` - Утилита для управления межсетевым экраном Netfilter, встроенным в ядро Linux. Позволяет создавать правила фильтрации сетевого трафика.
    *   `nftables` - Современная система фильтрации пакетов, призванная заменить `iptables`. Предоставляет более гибкий и мощный набор функций.
    *   `firewalld` - Динамический демон управления межсетевым экраном, использующий зоны и сервисы для упрощения настройки.

*   **Туннелирование и VPN:**
    *   `OpenVPN` - Популярное решение для создания защищенных VPN-соединений.
    *   `WireGuard` - Современный, высокопроизводительный и безопасный VPN-протокол.
    *   `ssh` - Может использоваться для создания зашифрованных туннелей (SSH-туннелирование).

**3. Удаленный доступ:**

*   `ssh` (Secure Shell) - Протокол и набор утилит для безопасного удаленного доступа к системам. Обеспечивает шифрование трафика и аутентификацию пользователей.
    *   `ssh` - клиент для подключения к удаленным серверам.
    *   `sshd` - демон, реализующий SSH-сервер.
    *   `ssh-keygen` - утилита для генерации ключей SSH.
    *   `ssh-agent` - агент для хранения ключей SSH в памяти.
*   `scp` (Secure Copy) - Утилита для безопасного копирования файлов между хостами по протоколу SSH.
*   `sftp` (Secure File Transfer Protocol) - Протокол и набор утилит для безопасной передачи файлов, аналогичный FTP, но работающий поверх SSH.
*   `rsync` - Утилита для синхронизации файлов и каталогов между локальными и удаленными системами. Может использовать SSH для безопасной передачи данных.
*   `telnet` (устаревает) - Протокол и утилита для удаленного доступа к системам. Не обеспечивает шифрование трафика, поэтому использовать его не рекомендуется.

**4. Сетевые службы (Серверы):**

*   **Веб-серверы:**
    *   `Apache` - Один из самых популярных веб-серверов в мире.
    *   `Nginx` - Высокопроизводительный веб-сервер и обратный прокси-сервер.

*   **Почтовые серверы:**
    *   `Postfix` - Популярный почтовый сервер (MTA).
    *   `Sendmail` - Один из старейших почтовых серверов.
    *   `Dovecot` - IMAP и POP3 сервер.

*   **DNS-серверы:**
    *   `BIND` (Berkeley Internet Name Domain) - Наиболее распространенный DNS-сервер.

*   **FTP-серверы:**
    *   `vsftpd` (Very Secure FTP Daemon) - Безопасный и быстрый FTP-сервер.
    *   `ProFTPD` - Модульный и настраиваемый FTP-сервер.

*   **Файловые серверы:**
    *   `Samba` - Позволяет предоставлять доступ к файлам и принтерам для клиентов Windows по протоколу SMB/CIFS.
    *   `NFS` (Network File System) - Протокол и набор утилит для предоставления доступа к файлам и каталогам по сети.

**5. Прочие утилиты:**

*   **Загрузка файлов:**
    *   `wget` - Консольная утилита для загрузки файлов из сети по протоколам HTTP, HTTPS и FTP.
    *   `curl` - Мощная утилита для передачи данных по различным протоколам (HTTP, HTTPS, FTP, SFTP, SCP и т.д.). Часто используется для тестирования API и взаимодействия с веб-сервисами.

*   **Информация о доменах и IP-адресах:**
    *   `whois` - Утилита для получения информации о регистрации доменных имен и IP-адресов.

*   **Сетевые файловые системы:**
    *   `sshfs` - Позволяет монтировать удаленные каталоги через SSH.

*   **Мониторинг:**
    *   `iftop` - Отображает использование полосы пропускания сетевыми интерфейсами в реальном времени.
    *   `nethogs` - Показывает, какие процессы используют сетевое соединение и сколько трафика они потребляют.

**1. Инструменты для мониторинга и анализа трафика:**

    **1.1. tcpdump:**
        *   **Основы синтаксиса:**
            *   Захват трафика с определенных интерфейсов.
            *   Фильтрация по IP-адресам, портам, протоколам (TCP, UDP, ICMP).
            *   Использование булевых операторов (and, or, not).
            *   Запись трафика в файлы pcap для последующего анализа.
            *   Чтение данных из файлов pcap.
        *   **Продвинутые техники фильтрации:**
            *   Фильтрация по содержимому пакетов (payload).
            *   Фильтрация по флагам TCP (SYN, ACK, FIN, RST, PSH, URG).
            *   Фильтрация по MAC-адресам.
            *   Использование BPF (Berkeley Packet Filter).
        *   **Анализ различных протоколов:**
            *   HTTP, HTTPS, FTP, DNS, SMTP, POP3, IMAP, SSH.
            *   Разбор заголовков и данных протоколов.
        *   **Интеграция с другими инструментами:**
            *   Wireshark, tshark.
        *   **Скриптинг и автоматизация:**
            *   Использование tcpdump в bash-скриптах.

    **1.2. Wireshark/tshark:**
        *   **Графический интерфейс Wireshark:**
            *   Навигация по захваченному трафику.
            *   Детальный просмотр пакетов и их содержимого.
            *   Фильтры отображения.
            *   Статистика по протоколам и эндпоинтам.
            *   Анализ потоков данных (follow TCP stream).
            *   Графическое представление данных.
            *   Экспорт данных в различные форматы.
        *   **Интерфейс командной строки tshark:**
            *   Автоматизированный анализ трафика.
            *   Извлечение специфичных данных из pcap файлов.
            *   Интеграция с другими инструментами и скриптами.
            *   Использование tshark для создания собственных инструментов анализа трафика.
        *   **Глубокий разбор протоколов:**
            *   Декодирование зашифрованного трафика (при наличии ключей).
            *   Анализ VoIP трафика.
            *   Анализ промышленных протоколов (Modbus, DNP3, etc.).
        *   **Lua скрипты:**
            *   Расширение функциональности Wireshark/tshark.
            *   Создание собственных диссекторов протоколов.
        *   **Анализ безопасности:**
            *   Выявление аномалий в трафике.
            *   Поиск вредоносного ПО.
        *   **pcapng формат:**
            *   Преимущества и особенности нового формата.

    **1.3. ngrep:**
        *   **Поиск по содержимому пакетов:**
            *   Использование регулярных выражений для поиска.
            *   Фильтрация по протоколам и портам.
            *   Отображение контекста найденных совпадений.
        *   **Сравнение с tcpdump и grep:**
            *   Когда использовать ngrep вместо tcpdump или grep.
        *   **Скриптинг:**
            *   Использование ngrep в скриптах для автоматизации поиска по сетевому трафику.

    **1.4. ntopng:**
        *   **Мониторинг трафика в реальном времени:**
            *   Веб-интерфейс для визуализации сетевой активности.
            *   Отображение активных хостов, протоколов, приложений.
            *   Графики использования полосы пропускания.
        *   **Исторический анализ:**
            *   Сохранение данных о трафике для последующего анализа.
            *   Построение отчетов по различным параметрам.
        *   **Геолокация:**
            *   Определение местоположения хостов по IP-адресам.
        *   **Алерты и уведомления:**
            *   Настройка оповещений при обнаружении подозрительной активности.
        *   **Интеграция с nProbe:**
            *   Сбор данных с удаленных устройств.

    **1.5. iftop:**
        *   **Мониторинг использования полосы пропускания в реальном времени:**
            *   Отображение текущего трафика по хостам и портам.
            *   Сортировка по объему трафика.
        *   **Фильтрация:**
            *   Отображение трафика только для определенных хостов или портов.

    **1.6. nload:**
        *   **Визуализация нагрузки на сетевой интерфейс:**
            *   Отображение входящего и исходящего трафика в виде графиков.
            *   Отображение средней и максимальной скорости.

    **1.7. bmon:**
        *   **Мониторинг пропускной способности различных сетевых интерфейсов**
            * Подробная статистика по каждому интерфейсу
            * Различные методы вывода
            * Поддержка разных юнитов измерения
            * Интерактивный режим
        *   **Настройка и фильтрация**
            * Выбор конкретных интерфейсов для мониторинга
            * Исключение определенных интерфейсов
        *   **Дополнительные модули**
            * Расширение функциональности для специфических задач

**2. Инструменты для диагностики сети:**

    **2.1. ping:**
        *   **Основы использования:**
            *   Проверка доступности хоста.
            *   Измерение времени отклика (RTT).
            *   Выявление потерь пакетов.
        *   **Продвинутые опции:**
            *   Изменение размера пакетов.
            *   Установка TTL (Time To Live).
            *   Указание интервала между запросами.
            *   Непрерывный пинг.
            *   Использование IPv6.
        *   **Flood ping:**
            *   Осторожное использование для тестирования производительности сети.
        *   **Интерпретация результатов:**
            *   Анализ времени отклика и потерь пакетов.
            *   Выявление проблем с маршрутизацией.

    **2.2. traceroute/tracepath:**
        *   **Определение маршрута следования пакетов:**
            *   Отображение промежуточных узлов (роутеров) на пути к целевому хосту.
            *   Измерение времени отклика до каждого узла.
        *   **Различия между traceroute и tracepath:**
            *   Использование разных протоколов (UDP, ICMP, TCP).
            *   Определение MTU (Maximum Transmission Unit).
        *   **Продвинутые опции:**
            *   Указание исходного порта.
            *   Установка максимального количества прыжков.
            *   Использование IPv6.
        *   **Визуализация маршрута:**
            *   Использование mtr (My Traceroute).
        *   **Анализ проблем с маршрутизацией:**
            *   Выявление узких мест в сети.
            *   Обнаружение петель маршрутизации.

    **2.3. mtr (My Traceroute):**
        *   **Комбинация ping и traceroute:**
            *   Непрерывное отображение статистики по каждому узлу в маршруте.
            *   Выявление потерь пакетов и задержек на каждом этапе.
        *   **Интерактивный режим:**
            *   Сортировка по различным параметрам.
            *   Настройка интервала обновления данных.
        *   **Экспорт данных:**
            *   Сохранение результатов в текстовый файл или CSV.

    **2.4. nslookup/dig:**
        *   **Основы работы с DNS:**
            *   Получение IP-адреса по доменному имени.
            *   Получение информации о DNS-серверах.
            *   Запросы различных типов записей (A, AAAA, MX, CNAME, PTR, SOA, NS, TXT).
        *   **Различия между nslookup и dig:**
            *   dig предоставляет более детальную информацию.
            *   dig более удобен для скриптинга.
        *   **Продвинутые возможности dig:**
            *   Указание конкретного DNS-сервера для запроса.
            *   Отладка DNS-запросов.
            *   Рекурсивные и нерекурсивные запросы.
            *   Обратные DNS-запросы (reverse lookup).
            *   Пакетный режим работы.
            *   Поддержка EDNS (Extension Mechanisms for DNS).
        *   **Анализ проблем с DNS:**
            *   Выявление проблем с разрешением имен.
            *   Проверка корректности настройки DNS-серверов.

    **2.5. host:**
        *   **Простые DNS-запросы:**
            *   Быстрое получение IP-адреса по доменному имени.
            *   Получение имени хоста по IP-адресу.
        *   **Сравнение с nslookup и dig:**
            *   Менее функциональный, но более простой инструмент.

    **2.6. arp:**
        *   **Работа с ARP-таблицей:**
            *   Просмотр и изменение ARP-таблицы.
            *   Сопоставление IP-адресов и MAC-адресов.
        *   **ARP-spoofing:**
            *   Понимание принципов атаки и методов защиты.
        *   **Gratuitous ARP:**
            *   Обновление ARP-таблиц других устройств в сети.

    **2.7. ip (iproute2):**
        *   **Управление сетевыми интерфейсами:**
            *   Просмотр информации об интерфейсах (ip addr).
            *   Включение/отключение интерфейсов (ip link set dev up/down).
            *   Настройка IP-адресов (ip addr add/del).
            *   Управление состоянием сетевых интерфейсов (up, down, promisc).
        *   **Управление таблицей маршрутизации:**
            *   Просмотр таблицы маршрутизации (ip route).
            *   Добавление/удаление маршрутов (ip route add/del).
            *   Настройка маршрута по умолчанию.
            *   Политики маршрутизации (policy routing).
        *   **Управление ARP-таблицей:**
            *   Просмотр и изменение ARP-таблицы (ip neigh).
        *   **Управление туннелями:**
            *   Создание и настройка различных типов туннелей (GRE, IPIP, SIT).
        *   **Управление сетевыми пространствами имен (network namespaces):**
            *   Изоляция сетевых ресурсов.
            *   Создание виртуальных сетей.
            *   Тестирование сетевых конфигураций. (ip netns)
        *   **Мониторинг изменений сетевых интерфейсов и маршрутов**
            * `ip monitor`
        *   **Traffic Control (tc):**
            *   Управление трафиком
            *   QoS (Quality of Service)
            *   Ограничение полосы пропускания
            *   Эмуляция задержек и потерь пакетов

    **2.8. netstat/ss:**
        *   **Отображение сетевых соединений:**
            *   Активные соединения (TCP, UDP).
            *   Слушающие порты (listening).
            *   Состояния соединений (ESTABLISHED, TIME_WAIT, CLOSE_WAIT, etc.).
        *   **Различия между netstat и ss:**
            *   ss более современный и быстрый инструмент.
            *   ss предоставляет более детальную информацию.
        *   **Отображение статистики по протоколам:**
            *   Количество отправленных и полученных пакетов.
            *   Ошибки передачи данных.
        *   **Отображение информации о маршрутизации:**
            *   Таблица маршрутизации.
        *   **Отображение информации о сокетах:**
            *   Статистика по использованию сокетов.
            *   Поиск процессов, использующих сетевые соединения (с опцией -p)
        *   **Опции для фильтрации и форматирования вывода**
        *   **Мониторинг изменений в реальном времени**
            * Использование `watch` совместно с `ss` или `netstat`

    **2.9. route:**
        *   **Управление таблицей маршрутизации (устаревший инструмент):**
            *   Просмотр таблицы маршрутизации.
            *   Добавление/удаление маршрутов.
            *   Настройка маршрута по умолчанию.
        *   **Рекомендуется использовать `ip route` вместо `route`**

**3. Инструменты для удаленного доступа и управления:**

    **3.1. SSH (Secure Shell):**
        *   **Безопасное удаленное подключение:**
            *   Шифрование трафика.
            *   Аутентификация по паролю или ключам.
        *   **Удаленное выполнение команд:**
            *   Запуск команд на удаленном сервере.
        *   **Проброс портов (port forwarding):**
            *   Локальный проброс портов.
            *   Удаленный проброс портов.
            *   Динамический проброс портов (SOCKS proxy).
        *   **X11 forwarding:**
            *   Запуск графических приложений на удаленном сервере.
        *   **SCP (Secure Copy):**
            *   Безопасное копирование файлов между хостами.
        *   **SFTP (SSH File Transfer Protocol):**
            *   Безопасный протокол передачи файлов.
        *   **Аутентификация на основе ключей:**
            *   Генерация пары ключей (ssh-keygen).
            *   Копирование публичного ключа на удаленный сервер (ssh-copy-id).
        *   **Конфигурационный файл SSH (~/.ssh/config):**
            *   Настройка параметров подключения для различных хостов.
            *   Использование алиасов.
        *   **Агент SSH (ssh-agent):**
            *   Управление ключами SSH.
        *   **Multiplexing:**
            *   Использование одного TCP-соединения для нескольких сессий SSH.
        *   **Безопасность SSH:**
            *   Настройка файрвола.
            *   Отключение аутентификации по паролю.
            *   Использование нестандартного порта.

    **3.2. telnet:**
        *   **Удаленное подключение (небезопасное):**
            *   Передача данных в открытом виде.
        *   **Использование для тестирования сетевых сервисов:**
            *   Проверка доступности порта.
            *   Отправка простых команд.
        *   **Не рекомендуется использовать для удаленного управления из-за отсутствия шифрования.**

    **3.3. nc (netcat):**
        *   **"Швейцарский нож" для работы с сетью:**
            *   Чтение и запись данных в сетевые соединения (TCP/UDP).
            *   Сканирование портов.
            *   Проброс портов.
            *   Создание простых чат-серверов.
            *   Передача файлов.
            *   Использование в качестве прокси.
            *   Слушающий режим (сервер) и режим подключения (клиент).
        *   **Скриптинг:**
            *   Использование netcat в скриптах для автоматизации сетевых задач.
        *   **Варианты netcat:**
            *   Традиционный netcat.
            *   Ncat (из состава Nmap).
            *   OpenBSD netcat.
            *   GNU netcat.
        *   **Примеры использования для тестирования и отладки.**
            * Создание простых веб-серверов
            * Бенчмаркинг сетевых сервисов

**4. Инструменты для сканирования сети:**

    **4.1. Nmap:**
        *   **Основы сканирования:**
            *   Определение активных хостов в сети (ping scan, ARP scan).
            *   Сканирование портов (TCP SYN scan, TCP connect scan, UDP scan).
            *   Определение операционной системы (OS fingerprinting).
            *   Определение версий сервисов (version detection).
        *   **Продвинутые техники сканирования:**
            *   Сканирование с использованием скриптов NSE (Nmap Scripting Engine).
            *   Обход файрволов и систем обнаружения вторжений (IDS evasion).
            *   Сканирование IPv6.
            *   Сканирование через прокси.
        *   **Скрипты NSE:**
            *   Использование готовых скриптов для различных задач.
            *   Написание собственных скриптов на Lua.
            *   Категории скриптов (auth, default, discovery, exploit, etc.).
        *   **Вывод результатов:**
            *   Сохранение результатов сканирования в различные форматы (XML, grepable, normal).
        *   **Интеграция с другими инструментами:**
            *   Zenmap (графический интерфейс для Nmap).
            *   Metasploit.
        *   **Юридические аспекты сканирования:**
            *   Сканирование только с разрешения владельца сети.
        *   **Сравнение с другими сканерами портов:**
            *   Unicornscan, Masscan.

    **4.2. hping3:**
        *   **Генерация и отправка произвольных сетевых пакетов:**
            *   Тестирование производительности сети.
            *   Тестирование файрволов.
            *   Создание флуда (DoS-атак, использовать с осторожностью).
        *   **Продвинутые возможности:**
            *   Установка различных флагов TCP.
            *   Фрагментация пакетов.
            *   Сканирование портов.
            *   Определение операционной системы.
        *   **Сравнение с Nmap:**
            *   hping3 более низкоуровневый инструмент.

**5. Инструменты для настройки сети:**

    **5.1. ethtool:**
        *   **Просмотр и изменение настроек сетевого интерфейса:**
            *   Скорость и дуплекс.
            *   Автосогласование (autonegotiation).
            *   Wake-on-LAN (WOL).
            *   Разгрузка аппаратного обеспечения (offloading).
            *   Статистика по ошибкам и коллизиям.
            *   Управление кольцевыми буферами
        *   **Диагностика проблем с сетевым интерфейсом.**
        *   **Тестирование кабеля**
            * `ethtool -t`

    **5.2. iwconfig/iw:**
        *   **Настройка беспроводных интерфейсов:**
            *   Просмотр информации о беспроводных сетях (iwlist scanning).
            *   Подключение к беспроводной сети (iwconfig essid key).
            *   Настройка режима работы (managed, ad-hoc, monitor).
            *   Установка мощности сигнала.
        *   **Различия между iwconfig и iw:**
            *   iw более современный инструмент.
            *   iw предоставляет больше возможностей.
        *   **wpa_supplicant:**
            *   Подключение к WPA/WPA2 защищенным сетям.
        *   **hostapd:**
            *   Создание точки доступа.

    **5.3. systemd-networkd:**
        *   **Управление сетевыми конфигурациями в systemd:**
            *   Настройка статических и динамических IP-адресов.
            *   Настройка DNS-серверов.
            *   Управление сетевыми мостами и VLAN.
            *   Настройка туннелей.
        *   **Конфигурационные файлы:**
            *   .network, .netdev, .link файлы.
        *   **networkctl:**
            *   Утилита для управления и просмотра состояния сети.

    **5.4. NetworkManager:**
        *   **Управление сетевыми подключениями:**
            *   Автоматическое подключение к известным сетям.
            *   Поддержка различных типов подключений (Ethernet, Wi-Fi, VPN, mobile broadband).
        *   **nmcli:**
            *   Интерфейс командной строки для управления NetworkManager.
            *   Просмотр и изменение сетевых настроек.
            *   Управление соединениями.
        *   **nmtui:**
            *   Текстовый интерфейс пользователя для управления NetworkManager.
        *   **Графические апплеты:**
            *   Интеграция с рабочими средами GNOME, KDE, etc.

            *   Шифрование трафика.
            *   Сжатие трафика.
        *   **Конфигурационные файлы:**
            *   .ovpn файлы.
        *   **Управление OpenVPN-сервером и клиентами.**
        *   **Интеграция с systemd**

    **6.2. WireGuard:**
        *   **Современный и высокопроизводительный VPN-протокол:**
            *   Простота настройки.
            *   Высокая скорость.
            *   Надежная криптография.
        *   **Конфигурационные файлы:**

**7. Файрволы:**

    **7.1. iptables (устаревший):**
        *   **Основы фильтрации трафика:**
            *   Таблицы (filter, nat, mangle, raw).
            *   Цепочки (INPUT, OUTPUT, FORWARD, PREROUTING, POSTROUTING).
            *   Правила (rules).
            *   Действия (ACCEPT, DROP, REJECT, LOG).
        *   **Фильтрация по IP-адресам, портам, протоколам.**
        *   **NAT (Network Address Translation):**
            *   SNAT (Source NAT).
            *   DNAT (Destination NAT).
            *   Masquerading.
        *   **Сохранение и восстановление правил.**
        *   **Модули:**
            *   conntrack (отслеживание соединений).
            *   limit (ограничение количества пакетов).

    **7.2. nftables:**
        *   **Современная замена iptables:**
            *   Более гибкий и производительный.
            *   Улучшенный синтаксис.
            *   Поддержка атомарных операций.
        *   **Основы работы с nftables:**
            *   Таблицы, цепочки, правила.
            *   Наборы (sets).
            *   Словари (maps).
        *   **Миграция с iptables на nftables.**
        *   **Интеграция с systemd.**

    **7.3. firewalld:**
        *   **Динамическое управление файрволом:**
            *   Зоны (zones).
            *   Сервисы (services).
            *   Богатые правила (rich rules).
        *   **firewall-cmd:**
            *   Интерфейс командной строки для управления firewalld.
        *   **Интеграция с NetworkManager.**

    **7.4. UFW (Uncomplicated Firewall):**
        *   **Простой в использовании интерфейс для iptables/nftables:**
            *   Упрощенный синтаксис.
            *   Настройка правил по умолчанию.
            *   Поддержка профилей приложений.

**9. BPF (Berkeley Packet Filter) и eBPF (extended BPF):**

    **9.1. Основы BPF:**
        *   Виртуальная машина для фильтрации пакетов в ядре.
        *   Использование в tcpdump и других инструментах.
        *   Синтаксис BPF.

    **9.2. eBPF:**
        *   Расширенная версия BPF.
        *   Выполнение произвольного кода в ядре.
        *   Трассировка, профилирование, сетевая фильтрация.

    **9.3. Инструменты для работы с eBPF:**
        *   **bpftrace:**
            *   Язык высокого уровня для написания eBPF программ.
            *   Трассировка системных вызовов, функций ядра, пользовательских приложений.
        *   **bcc (BPF Compiler Collection):**
            *   Набор инструментов и библиотек для разработки eBPF программ.
            *   Примеры использования eBPF для решения различных задач.
        *   **libbpf:**
            *   Библиотека для загрузки и управления eBPF программами.
